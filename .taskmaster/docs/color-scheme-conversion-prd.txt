# Color Scheme Conversion PRD

## Project Overview
Convert entire Ship Fast TS app to match the exact color scheme and design system from the TimeBack reference repository (https://github.com/dillonmurti/timeback-official-0b5aa02b.git).

## Design System Requirements

### Color Scheme
- **Primary Background**: #1abeff (light blue) - used for main page backgrounds
- **Primary Text/Elements**: #0f33bb (dark blue) - used for text, buttons, icons
- **Accent/White**: #ffffff - used for button backgrounds and contrast elements
- **Secondary decorative**: Blue and purple gradients for background decorations

### Typography
- **Primary Font**: Cal Sans (font-cal)
- **Secondary Font**: Satoshi (font-satoshi) 
- **Tertiary Font**: Crimson Pro (font-crimson)

### Design Patterns
- Clean, minimal two-color design system
- Rounded corners: 12px (rounded-xl)
- Subtle animations and transitions
- High contrast between light blue backgrounds and dark blue elements
- White buttons with dark blue text for CTAs

## Implementation Requirements

### 1. CSS Variables & Tailwind Config
- Update all CSS custom properties to match reference color values
- Modify Tailwind config to use new color system
- Ensure both light and dark mode support

### 2. Font Integration
- Add Cal Sans, Satoshi, and Crimson Pro fonts to the project
- Update font-family definitions in Tailwind config
- Apply font-cal as primary throughout the app

### 3. Component Updates
- Update all UI components to use new color scheme
- Replace all red/green colors with blue variants
- Update all gradients to match reference design
- Ensure consistent application of new colors

### 4. Page-by-Page Conversion
- Convert homepage/landing page
- Convert quiz flow pages
- Convert personalized experience pages
- Convert AI experience pages
- Convert dashboard and authentication pages
- Convert all modals, popups, and interactive elements

### 5. Brand Consistency
- Maintain consistent blue theme across all user touchpoints
- Ensure accessibility standards are met with new color contrast
- Update loading states and animations to match reference
- Apply rounded-xl styling consistently

## Technical Specifications

### CSS Variables to Update
```css
--background: #1abeff
--foreground: #0f33bb
--primary: #0f33bb
--primary-foreground: #ffffff
--secondary: #ffffff  
--secondary-foreground: #0f33bb
```

### Font Implementation
```css
font-family: {
  'cal': ['Cal Sans', 'sans-serif'],
  'satoshi': ['Satoshi', 'sans-serif'],
  'crimson': ['Crimson Pro', 'serif'],
}
```

### Forbidden Elements
- No red colors (text-red-*, bg-red-*, border-red-*)
- No green colors (text-green-*, bg-green-*, border-green-*)
- No gradients except subtle blue/purple background decorations
- No rainbow colors or multiple color schemes

## Success Criteria
- All pages match the reference design's color scheme exactly
- Consistent application of Cal Sans typography
- Maintained functionality while updated design
- Accessibility compliance with new color contrast
- No remaining red/green color elements
- Professional, cohesive blue theme throughout entire application

## Technical Constraints
- Must maintain existing functionality
- Must preserve responsive design
- Must work in both light and dark modes
- Must meet WCAG accessibility standards
- Must not break any existing components or features
